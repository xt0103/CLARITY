**CSV解析核心算法（parseLine函数）**：
```
1. 初始化：result数组、current字符串、inQuotes标志
2. 遍历每个字符：
   - 遇到引号：处理转义引号（""）或切换引号状态
   - 遇到逗号且不在引号内：将current加入result，清空current
   - 其他字符：追加到current
3. 将最后的current加入result
4. 返回result数组
```

| FR-019 | 匹配评分机制 | 为每个匹配计算分数 | utils/formDetector.js | autoMatch() | 遍历所有字段和CSV列，计算匹配分数，记录最高分 | field, csvHeader | number (分数) | 无 |
| FR-020 | 匹配阈值 | 设置最低匹配分数阈值 | utils/formDetector.js | autoMatch() | 只有分数≥20才建立映射关系 | bestScore | boolean | 无 |
| FR-021 | 重复映射避免 | 确保一个CSV列只映射一个字段 | utils/formDetector.js | autoMatch() | 使用Set记录已使用的CSV列，跳过已映射的列 | usedHeaders Set | 无重复映射 | 无 |
| FR-022 | 映射列表显示 | 在设置页面显示映射关系 | options.js | displayMappings() | 为每个字段创建下拉选择框，显示当前映射的CSV列 | detectedFormFields, fieldMappings | 更新DOM | 无字段或数据时显示提示 |
| FR-023 | 手动映射调整 | 允许用户手动选择CSV列映射 | options.js | displayMappings() | select元素的change事件，更新fieldMappings对象 | select.value | 更新fieldMappings | 空值删除映射 |
| FR-024 | 映射清除 | 清除单个或全部映射 | options.js | handleClearMappings() | 清空fieldMappings对象，重新显示映射列表 | 无 | 更新DOM和存储 | 确认对话框 |
| FR-025 | 映射配置保存 | 保存映射关系到本地存储 | options.js | handleSave() | 调用chrome.storage.local.set保存fieldMappings | fieldMappings | Promise | try-catch错误处理 |

**自动匹配算法详细流程**：
```
1. 初始化：复制existingMappings，创建usedHeaders Set
2. 遍历每个表单字段：
   a. 如果已有映射，跳过
   b. 收集匹配文本（name、id、placeholder、label转小写）
   c. 遍历每个CSV列：
      - 如果列已使用，跳过
      - 计算匹配分数：
        * 精确匹配：+100分
        * 包含匹配：+50分
        * 关键词匹配：+30分
        * 部分匹配：+10分
      - 记录最高分和最佳匹配
   d. 如果分数≥20，建立映射并标记列已使用
3. 返回映射对象
```

**关键词匹配逻辑**：
```
1. 检查字段标识中是否包含关键词（keywordMap中的任一关键词）
2. 检查CSV列名中是否包含相同关键词
3. 如果两者都包含，+30分
```

**字段填写详细逻辑**：

**文本输入框（text/email/tel/number等）**：
```
element.value = trimmedValue
触发input和change事件
```

**复选框（checkbox）**：
```
判断value是否为'true'/'1'/'yes'/'on'/'checked'
element.checked = shouldCheck
```

**单选框（radio）**：
```
遍历所有radio，匹配value
匹配成功：element.checked = true
```

**下拉选择框（select）**：
```
1. 精确匹配option.value（不区分大小写）
2. 精确匹配option.textContent（不区分大小写）
3. 部分匹配（包含关系）
设置element.selectedIndex
```

**多行文本（textarea）**：
```
element.value = trimmedValue
```

**填写流程**：
```
1. 验证数据有效性（csvData存在且rowIndex有效）
2. 获取指定行的数据
3. 遍历所有字段：
   a. 查找映射的CSV列
   b. 获取数据值
   c. 查找DOM元素
   d. 调用fillField填写
   e. 触发事件
   f. 统计成功数量
4. 返回结果对象
```

## 10. 技术实现细节

### 10.1 CSV解析算法
- **引号处理**：支持RFC 4180标准，处理转义引号（""）
- **分隔符**：使用逗号（,）作为分隔符
- **编码**：强制使用UTF-8编码读取
- **空值处理**：空单元格存储为空字符串

### 10.2 字段选择器生成策略
1. **优先级1**：`#id` - 最快，最准确
2. **优先级2**：`[name="xxx"]` - 较准确
3. **优先级3**：`tag:nth-of-type(n)` - 兜底方案

### 10.3 匹配算法评分规则
- **精确匹配**：100分（完全一致）
- **包含匹配**：50分（一方包含另一方）
- **关键词匹配**：30分（都包含相同关键词）
- **部分匹配**：10分（单词匹配）
- **阈值**：≥20分才建立映射

### 10.4 事件触发机制
- 使用原生Event构造函数
- 设置bubbles: true（冒泡）
- 设置cancelable: true（可取消）
- 触发input和change两个事件
